## Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.** 

If you discover a security vulnerability within this repository, please report it through one of the following secure channels:

* **Email:** Send a detailed report to [1st Author](mailto:anumitasam@gmail.com).
* **GitHub Private Vulnerability Reporting:** If enabled for this repository, please utilize the "Report a vulnerability" button under the **Security** tab.

### What to Include in Your Report:
To help our internal team address the issue as quickly as possible, please include:
* A description of the vulnerability and its potential impact.
* Detailed steps to reproduce the issue (or a proof-of-concept script).
* Any specific configurations required to trigger the vulnerability.

## Our Process

* **Acknowledgement:** A chosen collaborator will acknowledge receipt of your report within 7 days.
* **Evaluation:** Our team will investigate the issue and determine the appropriate fix.
* **Resolution:** We will work on a patch or mitigation strategy. Because we control all write access to this repository, a fix will be pushed directly by an authorized collaborator once it is ready.
* **Disclosure:** We will coordinate with the reporter regarding public disclosure timelines if applicable.

Thank you for helping keep this project secure!
